## 1. 项目概述

### 1.1 产品定位
一个面向教育内容创作者的轻量级工具，用于快速生成教学视频中的语法对比图。主要服务于英语语法教学场景，帮助创作者在3秒内向学生传达清晰的语法概念对比。

### 1.2 目标用户
- **主要用户**: 英语教学视频创作者
- **使用场景**: 制作8-11岁小学生的英语语法课程视频
- **技术水平**: 具备基本的计算机操作能力，无需编程背景

### 1.3 核心价值
- **效率提升**: 从"手动PS/Canva设计30分钟" → "配置生成30秒"
- **质量保证**: 统一的视觉风格，避免设计不一致
- **快速迭代**: 一键生成多个版本，快速选择最佳方案

### 1.4 成功指标
- 单个对比图生成时间 < 1分钟
- 视觉质量满足视频使用标准（清晰度、可读性）
- 用户满意度："这个就挺好" / "格式很好"

## 2. 用户研究

### 2.1 用户画像
**用户：Archie**
- 身份：研究生 + 教育内容创作者
- 需求：每周制作2-3个语法教学视频
- 痛点：
  - 每次需要设计语法对比图，重复劳动
  - AI生成的图片质量不稳定（字体乱码、风格不统一）
  - 需要快速试错，找到最合适的视觉表达

### 2.2 使用流程
```
创作者工作流：
1. 编写课程脚本 
2. 识别需要视觉辅助的知识点（如：must vs have to）
3. 【使用本工具】生成对比图
4. 截图/录屏
5. 导入视频编辑软件（剪映/PR）
6. 叠加到讲解画面上（占半屏，停留3秒）
```

### 2.3 竞品分析

| 工具 | 优点 | 缺点 | 适用性 |
|------|------|------|--------|
| Canva | 模板丰富 | 需要手动调整，效率低 | ❌ |
| Figma | 专业设计工具 | 学习成本高，过度设计 | ❌ |
| PPT | 熟悉易用 | 每次从零开始，重复劳动 | ⚠️ |
| AI生成（Gemini/Midjourney） | 快速 | 质量不可控，中文乱码 | ❌ |
| **本工具** | 快速 + 质量可控 + 格式统一 | - | ✅ |

### 3.2 MVP功能清单（v1.0）

#### 3.2.1 配置面板
**必需字段**：
- [x] 配置名称：可自定义配置名称（如 "Must vs Have to"）
- [x] 布局方向：横向排列 / 纵向排列
- [x] 对比项配置（支持多个对比项，可拖拽排序）
  - 显示样式：图标样式 / 文本框样式
  - 标题（英文）：如 "must"（必填）
  - 副标题（中文）：如 "主观 - 我觉得"
  - 颜色选择：色块选择器或HEX输入
  - **图标样式配置**：
    - 形状选择：圆形/方形
    - Emoji列表：可添加/删除多个emoji，支持分类选择器
  - **文本框样式配置**：
    - 文本框内容：支持文字和emoji混排，支持样式标记
    - 文本框样式：圆角矩形/长方形
    - 框外Emoji：可在文本框外放置emoji
    - 框外Emoji位置：上方/下方/左侧/右侧

**交互要求**：
- 实时预览：修改配置后立即在预览区更新
- 表单验证：必填字段不能为空
- 默认值：提供一套默认配置（must vs have to）
- 拖拽排序：对比项可通过拖拽调整顺序

#### 3.2.2 预览区域
**显示内容**：
- 对比图预览（占据视口60-80%）
- 版本切换按钮（顶部，支持新增/删除版本）
- 当前版本指示器

**交互行为**：
- 点击版本按钮 → 切换emoji显示
- 高亮显示当前版本
- **位置自由调整**：
  - 支持拖拽对比项到任意位置
  - 每个版本的位置信息独立保存
  - 实时更新位置，自动保存
- **版本管理**：
  - 点击"+ 新增版本"按钮创建新版本
  - 鼠标悬停版本按钮显示删除按钮（×）
  - 至少保留一个版本

#### 3.2.3 版本管理
**版本定义**：
- 一个"版本"= 所有对比项的emoji组合 + 位置信息
- 示例：
  - 版本1: 🤔 思考 + 📏 尺子（位置A）
  - 版本2: 🤔 思考 + ❗ 感叹号（位置B）
  - 版本3: 🧠 脑子 + 📏 尺子（位置C）

**生成逻辑**：
- **自动生成**：一键生成所有可能的emoji组合
- **手动管理**：支持手动新增/删除版本
- **位置信息**：每个版本的位置信息独立保存
- **版本命名**：自动命名或手动编辑

**版本操作**：
- [x] 一键生成所有版本
- [x] 手动新增版本
- [x] 删除版本（至少保留一个）
- [x] 版本切换预览

#### 3.2.4 文字样式功能
**文本框样式标记**：
- [x] **文字颜色**：`[color:#ff0000]文字[/color]` - 改变文字颜色（支持十六进制和颜色名）
- [x] **高亮背景**：`[highlight:#ffff00]文字[/highlight]` - 设置文字背景色
- [x] **加粗**：`[bold]文字[/bold]` - 文字加粗显示
- [x] **斜体**：`[italic]文字[/italic]` - 文字斜体显示
- [x] **下划线**：`[underline]文字[/underline]` - 文字下划线
- [x] **删除线**：`[strike]文字[/strike]` - 文字删除线
- [x] **自定义样式**：`[style:color:#ff0000;background:#ffff00]文字[/style]` - 支持任意CSS样式

**使用示例**：
```
must [color:#ff0000]必须[/color] [highlight:#ffff00]强调[/highlight] [bold]重要[/bold]
```

**功能特点**：
- 实时预览：输入后立即在预览区看到效果
- 样式组合：可同时使用多种样式
- 导出支持：导出的HTML文件保留所有样式
- 样式说明：配置面板中提供使用说明

#### 3.2.5 导出功能
- [x] **HTML导出**：生成独立的HTML文件，包含所有样式和位置信息，可直接在浏览器打开
- [x] **配置导出**：保存为JSON文件，包含版本和位置信息，方便复用
- [x] **配置保存**：自动保存到LocalStorage，页面刷新后自动加载

## 4. 技术方案

### 4.1 技术栈对比

#### 方案A：纯前端（推荐 - MVP快速上线）
**技术栈**：
- HTML + CSS + Vanilla JavaScript
- 或 React + Tailwind CSS

**优点**：
- ✅ 开发速度快（1-2天）
- ✅ 无需后端，部署简单（GitHub Pages）
- ✅ 用户无需注册，直接使用
- ✅ 离线可用

**缺点**：
- ❌ 无法多设备同步
- ❌ 配置只能本地保存
- ❌ 无法分享给他人

**适用性**：✅ **MVP首选**

---

#### 方案B：Next.js + Supabase（推荐 - 长期发展）
**技术栈**：
- 前端：Next.js 14 + TypeScript + Tailwind CSS
- 后端：Supabase（数据库 + Auth + Storage）

**优点**：
- ✅ 支持用户账号

- ✅ 可分享配置链接

**缺点**：
- ⚠️ 开发时间较长（1周）

**适用性**：✅ **长期推荐**

- Electron + React

**优点**：
- ✅ 原生应用体验

**缺点**：
- ❌ 开发复杂度高
- ❌ 跨平台打包麻烦
- ❌ 过度设计（对于当前需求）

### 4.3 核心数据模型

#### ComparisonConfig（对比图配置）
```typescript
interface ComparisonConfig {
  name: string;                  // 配置名称（如"Must vs Have to"）
  layout: 'horizontal' | 'vertical'; // 布局方向
  items: ComparisonItem[];       // 对比项列表
  versions: Version[];           // 版本列表
  manualVersions: boolean;       // 是否手动管理模式
  createdAt: string;             // ISO日期字符串
  updatedAt: string;             // ISO日期字符串
}

interface ComparisonItem {
  id: number;                    // 唯一标识
  title: string;                 // 英文标题（必填）
  subtitle: string;              // 中文副标题
  color: string;                 // HEX颜色
  displayStyle: 'icon' | 'textbox'; // 显示样式
  // 图标样式配置
  shape: 'circle' | 'square';    // 形状
  emojis: string[];              // emoji列表
  // 文本框样式配置
  boxContent: string;            // 文本框内容（支持样式标记）
  boxStyle: 'rounded' | 'rectangle'; // 文本框样式
  externalEmoji: string;         // 框外emoji
  externalEmojiPosition: 'top' | 'bottom' | 'left' | 'right'; // 框外emoji位置
}

interface EmojiVersion {
  id: string;
  emoji: string;                 // emoji字符
  label: string;                 // 版本名称（如"尺子"）
}

interface Version {
  name: string;                  // 版本名称（如"版本1"）
  combination: string[];          // emoji组合数组
  positions: {                   // 位置信息
    [itemIndex: number]: {       // 每个item的位置
      x: number;                 // X坐标
      y: number;                 // Y坐标
    };
  };
}
```

## 8. 风险和挑战

### 8.1 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| Emoji跨平台显示不一致 | 中 | 高 | 使用Emoji字体库（如Noto Emoji）或SVG替代 |
| HTML导出后样式丢失 | 高 | 中 | 使用内联样式，避免外部CSS依赖 |
| 浏览器兼容性问题 | 中 | 低 | 使用Autoprefixer，目标浏览器限定为现代浏览器 |
| 截图功能性能问题 | 低 | 中 | 使用Web Worker处理，或改为服务端生成 |

### 8.2 产品风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 用户需求变化 | 高 | 中 | 快速迭代，保持MVP思维 |
| 功能过度设计 | 中 | 高 | 严格按照PRD，避免Feature Creep |
| 用户学习成本高 | 高 | 低 | 提供视频教程和默认配置 |

### 8.3 时间风险
- **开发延期**: 采用敏捷开发，每个Sprint独立交付可用版本
- **测试不足**: MVP阶段自己使用，边用边改
